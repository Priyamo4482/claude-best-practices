# Contributing

Found something that belongs here? Open a PR.

## What makes a good contribution
- Actionable - "do X when Y" beats "X is important"
- Tested - only add what you've seen work in practice
- Sourced - credit the original author where possible
- Placed correctly - add to the right file, update 00-index.md

## How to contribute
1. Fork the repo
2. Add your content to the relevant file
3. Update `00-index.md` if you're adding a new scenario
4. Open a PR with a short description of what you added and where it came from
